# 12 Interaction · 交互规范

统一手势与反馈，让操作结果可预期。

## 反馈映射

| 手势 | 反馈 |
| --- | --- |
| 点击 Tap | Opacity 叠加 + Ripple 水波纹，`MotionInstant` (100ms) |
| 长按 Long Press | 触发上下文 Menu + 轻微 Scale(0.98) + 触感反馈 |
| 拖动 Drag | 元素 Scale(1.03) + `Level2` 浮起，跟手移动 |
| 滑动 Swipe | 跟手，松手 `EaseSpring` 回弹或吸附 |
| 下拉 Pull | 下拉刷新指示器，超过阈值触发 |
| 双指缩放 Pinch | 内容缩放，边界回弹 |

## 反馈原则

- **即时**：交互 100ms 内必须有视觉反馈。
- **可逆**：破坏性操作提供撤销（Undo），而非仅确认弹窗。
- **一致**：相同手势在全 App 结果一致（如左滑始终为返回，不作删除）。
- **触感**：关键操作（成功/错误/切换）配合系统触感（Haptic）。

## 状态反馈时间预期

| 耗时 | 反馈方式 |
| --- | --- |
| < 100ms | 无需 Loading |
| 100ms ~ 1s | 局部 Loading / 按钮加载态 |
| > 1s | 明确进度指示 / Skeleton |
| > 10s | 可取消 + 进度百分比 |

## 防误触

- 命中区 ≥ 44×44。
- 破坏性行动与主行动保持距离，不相邻放置。
- 危险操作使用 `Danger` 色 + 二次确认或滑动确认。
