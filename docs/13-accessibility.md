# 13 Accessibility · 无障碍

无障碍是底线，**必须写**。目标：任何人在任何情境下都能使用。

## 颜色对比度（WCAG）

| 内容 | 最低对比度 |
| --- | --- |
| 正文文字（< 18pt） | 4.5 : 1 |
| 大字文字（≥ 18pt Bold / 24pt） | 3 : 1 |
| 图标 / 边框等非文字 | 3 : 1 |

- 不能仅靠颜色传达信息（如错误需图标 + 文案，不只变红）。
- Token 的 Light/Dark 值均需通过对比度校验。

> 说明：完整的无障碍合规需借助辅助技术实测与专家评审，本规范提供设计约束，不替代人工验证。

## 触摸区域

- 所有可交互元素命中区 **≥ 44×44 vp**，即使视觉更小。
- 相邻可点元素间距 ≥ `SpaceXS` (8)。
- **紧凑密度（compact）**：控件视觉可小于 44，但必须用 `.responseRegion(...)` 把命中区兜底到 `Token.size.minTouch`(44)，不得因收紧而牺牲可点性（见 [16 Design Token](16-design-token.md) 的「密度 / Compact」）。

## 字体缩放

- 使用 `fp` 单位，支持系统字体放大至 200%。
- 布局需在大字号下不截断、不重叠（用弹性布局，避免固定高度装文字）。

## 屏幕阅读器（TalkBack / VoiceOver / Harmony Accessibility）

- 每个可交互元素提供语义 Label（`accessibilityText`）。
- 图标按钮必须有文字描述（如"返回""更多"）。
- 提供状态播报（选中/展开/加载中/错误）。
- 合理的焦点顺序，遵循视觉阅读顺序。
- 装饰性元素标记为忽略（`accessibilityLevel: "no"`）。

## 其他

- 支持"减弱动态效果"，降级动画（见 [Motion](08-motion.md)）。
- 表单错误明确关联字段并可被读出。
- 支持键盘 / 外接设备焦点导航（Focused 态清晰可见）。

## 检查清单

- [ ] 文字对比度达标
- [ ] 命中区 ≥ 44×44
- [ ] 所有图标按钮有 Label
- [ ] 信息不只靠颜色
- [ ] 大字号不破版
- [ ] 焦点顺序合理
