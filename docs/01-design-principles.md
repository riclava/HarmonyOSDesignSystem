# 01 Design Principles · 设计原则

设计原则是整套系统的价值观，用于在没有明确规范时做取舍。

## 五大原则

### 1. 简洁 Simple
减少视觉噪音。每个界面只强调一个主任务，弱化次要信息。
- 一屏一个主行动（Primary Action）。
- 默认留白，慎用分割线与边框。
- 能用层级（间距、字重）表达的，不用装饰。

### 2. 一致 Consistent
所有页面保持统一体验。相同的元素在任何地方行为一致。
- 相同语义使用相同 Token（颜色 / 间距 / 圆角）。
- 相同操作使用相同交互反馈。
- 跨 App 复用同一套组件与命名。

### 3. 高效 Efficient
操作路径尽可能短。让用户以最少步骤完成目标。
- 高频操作前置、可达。
- 提供合理默认值，减少输入。
- 避免不必要的确认与跳转。

### 4. 自然 Natural
符合 HarmonyOS 的交互习惯与物理直觉。
- 动效遵循真实世界的加减速。
- 手势方向与结果一致（下滑关闭、左滑返回）。
- 反馈即时，状态可见。

### 5. 智能 Smart
充分利用 AI 与系统能力，主动而不打扰。
- 预测意图，提供建议而非强制。
- 情境感知（时间、位置、设备形态）。
- 智能能力可关闭、可解释、可信任。

## 决策优先级

当原则冲突时，按以下顺序取舍：

```
可用性 (Accessibility) > 一致性 (Consistent) > 高效 (Efficient) > 简洁 (Simple) > 美观
```

无障碍与一致性是底线，不可为视觉效果牺牲。
